    Codespaces HB

    One of the most frustrating aspects of learning to code is having a properly set up working environment to run the code from the materials you are learning from. To counter this and further ease your transition into coding we have made use of GitHubs CodeSpaces, which can be utilised as a coding environment with all the components already installed.

    Codespaces are a virtual machine, a computer instance hosted in the cloud, that can run the Microsoft Visual Studio Code IDE for editing and executing code (see the Orientation material for more details on this tool). Behind the scenes, this virtual machine comes pre-installed with everything you need: Python, Jupyter support, the required libraries for the module, and version control via Git. This eliminates the need to install anything locally and ensures that all students are working in an identical, error-free environment.

    Codespaces also help us ensure fairness and consistency in grading. Once your work is complete, you'll submit it directly through Git's version control system - all from inside the Codespace interface.

    A Codespace can be created for each individual repository, however there are limits to how many you can have and for how long per month (see below).

    The video below will walk you through how to create a Codespace from your repository, including a guide on what to do with an error that occasionally occurs.

    Codespace limit

    A very common occurrence is a student running out of their alloted allowance of hours for the month for their Codespaces, rendering them unable to complete their work in a manner they are used to. This can be rectified by setting up Python and an IDE on your local computer, and connecting it to GitHub (see Python Orientation). However, this can be technical and confusing for beginners.

    Instead, we ask students to be aware of their usage to prevent the limit being reached. Each student is allocated 90 hours per month, which given approximately 2 lessons per month is 45 hours per lesson. This may sound like a lot, however these hours are still consumed if the Codespace is idle not stopped. To stop a Codespace close the tab where it is open, which will cease its activity after 30 minutes. Additionally, remember to delete your Codespace after you have finished your work (see Assignment Submission).

    See the box below for a summary of the do's and do not's:

    IMPORTANT

    Key points:

    • DO NOT leave the Codespace open in a tab

    • DO delete your Codespace after finishing a lesson or before a long break in activity

    • DO check the number of Codespaces you have open at www.github.com/codespaces
    You can always spin up a new Codespace if needed, so do not feel the need to keep one active between long bouts of inactivity!
